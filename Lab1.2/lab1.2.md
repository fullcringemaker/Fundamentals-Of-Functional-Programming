# Цель работы
Целью данной работы является изучение базовых объектно-ориентированных возможностей языка Scala.

# Индивидуальный вариант
Булевская формула, состоящая из вхождений имён переменных, а также операций И, ИЛИ и НЕ. Конструктор 
формулы принимает в качестве параметра имя переменной и порождает формулу, состоящую из вхождения этой 
переменной. Операции: объединение двух формул в одну с помощью операций И («*») и ИЛИ («+»); отрицание 
формулы («!»). Кроме того, у формулы должен быть метод «eval», принимающий в качестве параметра 
ассоциативный массив, отображающий имена переменных в их значения, и вычисляющий значение формулы.

# Реализация и тестирование

```scala
class BoolFormula private (f: Map[String, Boolean] => Boolean) {

  def this(name: String) = this(env => env(name))

  def eval(env: Map[String, Boolean]): Boolean = f(env)

  def + (g: BoolFormula): BoolFormula =
    new BoolFormula(env => this.eval(env) || g.eval(env))

  def * (g: BoolFormula): BoolFormula =
    new BoolFormula(env => this.eval(env) && g.eval(env))

  def unary_! : BoolFormula =
    new BoolFormula(env => !this.eval(env))
}
```

требовалось обеспечить возможность строить выражение из имени переменной, комбинировать выражения 
операциями И, ИЛИ и НЕ, а также вычислять итоговое значение по заданному набору значений переменных.

В ходе выполнения индивидуального варианта формула была реализована так, чтобы её внутреннее 
представление задавалось правилом вычисления, а построение более сложных выражений происходило путём 
комбинирования уже имеющихся правил. Благодаря этому операции над формулами не изменяют существующие
 объекты, а создают новые, которые при вычислении обращаются к значениям переменных из переданного 
окружения и применяют нужную логику к результатам частей выражения.

В результате получено решение, демонстрирующее базовые объектно-ориентированные возможности Scala: 
использование приватного первичного конструктора и вспомогательного конструктора, определение методов 
и перегрузка операторов для естественной записи выражений. Проведённое тестирование подтверждает 
корректность вычислений для различных наборов входных значений, а используемый подход демонстрирует 
применение базовых принципов ООП в Scala вместе с функциональной идеей “объект как обёртка над 
поведением”
